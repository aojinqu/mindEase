# MindEase 产品需求文档（PRD）

## 1. 产品概述

**产品名称**  
MindEase

**产品类型**  
Android 移动应用

**开发平台**  
Android Studio + Java

**目标用户**  
希望记录情绪、理解情绪变化、释放压力并获得轻量情绪支持的大学生

**产品定位**  
MindEase 是一款面向大学生的 AI 辅助情绪日记与心理支持 App。它结合每日情绪记录、情绪趋势分析、匿名表达空间和个性化建议，帮助用户更好地理解和管理自己的情绪状态。

**一句话介绍**  
一款帮助大学生记录心情、发现情绪规律并获得实用支持建议的轻量情绪陪伴 App。

---

## 2. 项目背景与动机

大学生常常面临学业压力、截止日期、社交关系、职业规划和个人生活等多方面压力。很多学生希望记录自己的情绪，或者寻找一个低压力的方式表达想法，但现有心理健康类应用往往存在以下问题：

- 过于医疗化或专业化
- 操作复杂，日常使用门槛高
- 更适合成年人或泛用户，而非学生群体
- 提供泛泛建议，缺乏可执行性

MindEase 的目标是：

- 提供简单快速的每日情绪记录方式
- 通过图表和日历帮助用户理解情绪变化
- 提供低压力、匿名的表达空间
- 基于 AI 或规则分析提供短小、实用、非医疗化的支持建议

本产品定位为“日常情绪陪伴工具”，而不是医学诊断工具。

---

## 3. 产品目标

### 3.1 主要目标

- 帮助用户以低门槛方式记录每日情绪
- 帮助用户通过可视化图表理解情绪趋势
- 提供轻量级 AI 情绪分析与自助建议
- 提供匿名、安全的表达空间
- 提高用户对自身情绪状态的觉察能力与管理习惯

### 3.2 非目标

- 不提供医疗诊断
- 不替代专业心理咨询或治疗
- 不在第一版中做成完整社交平台
- 不在第一版中支持 iOS

---

## 4. 目标用户

### 4.1 核心用户

- 本科生与研究生
- 面临学业压力、社交压力或情绪波动的学生
- 想记录心情但不想使用复杂心理健康工具的用户

### 4.2 使用场景

- 在一天结束后记录当天心情
- 在考试周或赶 ddl 期间查看自己的压力变化
- 想匿名倾诉，但不希望暴露真实身份
- 在心情低落或焦虑时获得简单可执行的建议
- 回顾自己一周或一个月的情绪变化

---

## 5. 核心价值

MindEase 与普通日记 App 的不同之处在于，它不只是“记录情绪”，而是进一步帮助用户：

- 记录情绪
- 理解情绪趋势
- 识别情绪触发因素
- 获得可执行的支持建议

产品强调“觉察 + 轻干预 + 可持续使用”，并以学生友好、低压力、非医疗化为设计原则。

## 7. 功能需求

## 7.1 用户账户模块

**功能说明**  
用户可创建账号、登录系统并管理基础个人信息。

**主要功能**

- 用户注册
- 用户登录 / 退出登录
- 基础资料设置
- 可选昵称 / 头像
- 隐私设置

**需求要求**

- 用户必须能够通过邮箱和密码完成注册
- 用户必须能够安全登录
- 系统必须将情绪日记与对应用户关联
- 匿名社区中不得显示真实身份信息

---

## 7.2 每日情绪记录模块

**功能说明**  
用户可通过表情图标、滑动选择、标签和文本来记录当前情绪。

**主要功能**

- 通过表情或图标选择心情
- 选择情绪强度，范围为 1 到 5
- 输入文字日记
- 选择情绪相关标签
- 支持保存、编辑、删除记录

**情绪类型示例**

- 开心
- 平静
- 焦虑
- 难过
- 生气
- 疲惫
- 孤独
- 压力大

**标签示例**

- 学业
- 考试
- 截止日期
- 友情
- 家庭
- 恋爱
- 健康
- 睡眠
- 金钱

**需求要求**

- 用户必须能在 3 步内完成一次情绪记录
- 用户必须能编辑和删除历史记录
- 每条记录必须包含时间戳
- 每条记录应至少包含情绪类型、情绪强度、可选文本和可选标签

---

## 7.3 AI 情绪分析模块

**功能说明**  
系统结合用户输入文本和历史记录，对情绪倾向与变化趋势进行分析。

**主要功能**

- 对日记文本进行情绪分析
- 判断情绪倾向为正向、中性或负向
- 生成周度与月度情绪总结
- 分析主要压力来源标签分布
- 用图表展示情绪变化趋势

**输出示例**

- “你本周的焦虑情绪主要与学业压力有关。”
- “与你上周相比，本周整体情绪更加稳定。”
- “带有睡眠标签的记录中，负面情绪出现频率更高。”

**需求要求**

- 系统至少支持基础文本情感分析
- 系统至少展示近 7 天和近 30 天的趋势
- 分析结果必须以可视化形式呈现
- 若 AI 或 API 分析失败，系统仍需提供规则分析或基础统计结果

**实现说明**

课程项目中可采用以下方式：

- 基于关键词词典的情感分析
- 基于规则的情绪分值计算
- 接入第三方情感分析 API

---

## 7.4 情绪趋势可视化模块

**功能说明**  
用户可通过图表查看自己在一段时间内的情绪变化。

**主要功能**

- 周度情绪折线图
- 月度情绪统计图
- 标签频率统计
- 正向 / 负向情绪占比图

**需求要求**

- 系统至少提供一个折线图和一个统计图
- 用户必须能够切换周视图与月视图
- 图表必须适合手机屏幕浏览

---

## 7.5 情绪日历模块

**功能说明**  
系统以日历形式展示用户每日情绪记录。

**主要功能**

- 在日历中展示每日情绪状态
- 使用颜色或图标区分不同情绪
- 点击日期查看当天详细记录
- 可选支持连续打卡显示

**需求要求**

- 系统必须提供日历视图
- 用户必须能够点击任意已记录日期查看详情
- 不同情绪类型应使用不同颜色或图标进行区分

---

## 7.6 个性化建议模块

**功能说明**  
系统根据近期情绪记录，为用户生成简短、可执行的自助支持建议。

**主要功能**

- 提供每日建议
- 给出减压建议
- 推荐运动、休息、呼吸练习、写作或社交相关小行动
- 根据近期趋势动态调整建议内容

**建议示例**

- “你已经连续 3 天感到压力较大，可以试试 2 分钟深呼吸。”
- “你今天可能有些社交疲惫，建议减少信息摄入并早点休息。”
- “你本周状态较稳定，可以继续保持当前的学习和休息节奏。”

**需求要求**

- 建议必须基于最近 3 到 7 天的记录生成
- 建议必须短小、具体、可执行
- 建议内容不得包含医疗诊断表述

---

## 7.7 匿名树洞社区模块

**功能说明**  
用户可匿名发布情绪想法，并浏览他人的匿名内容。

**主要功能**

- 匿名发帖
- 浏览社区内容流
- 按情绪标签筛选
- 点赞帖子
- 可选评论 / 回复
- 提供“抱抱你”“支持一下”等低压力互动按钮

**需求要求**

- 系统必须隐藏真实身份信息
- 用户必须能够发帖和浏览帖子
- 每条帖子应支持情绪标签
- 系统应具备基础内容审核规则
- 社区风格应保持支持性、低攻击性

---

## 7.8 心理疗愈 Agent 模块

**功能说明**  
系统提供一个面向用户的心理疗愈 Agent，对话过程中结合用户最新情绪记录、近期趋势、标签和建议等上下文，输出温和、支持性、非医疗化的陪伴式回应。

**主要功能**

- 提供独立的 Agent 对话入口
- 支持多轮对话与会话历史保留
- 在生成回复前自动注入用户最新数据
- 支持通过外部中转站接入第三方大模型 API
- 支持基于 Prompt 模板控制回复风格、安全边界与输出结构
- 当远程模型不可用时，降级为本地规则安抚文案或引导语

**用户最新数据示例**

- 最近一次情绪记录
- 最近 3 到 7 天情绪趋势
- 高频压力标签
- 最新个性化建议
- 当日对话中的最近几轮消息

**需求要求**

- Agent 回复必须明确保持非医疗化、非诊断定位
- 系统必须在发送 Prompt 前筛选并拼装用户最新数据
- 对话响应应尽量控制在 3 秒内返回首条结果
- 若外部 API 失败，系统必须提供可用的降级回复
- 系统必须保留基础安全策略，对高风险表达进行提醒与资源引导

## 8. 创新功能

以下功能将帮助 MindEase 与市面上同类应用形成差异化。

## 8.1 情绪触发因子地图

系统不仅展示用户的情绪结果，也分析导致负面或正面情绪的高频因素。

**示例**

- 焦虑常与“考试”和“睡眠不足”标签相关
- 难过常与“关系问题”和“孤独”标签相关

**价值**

帮助用户理解“为什么会这样”，而不是只知道“今天不开心”。

---

## 8.2 AI 安慰重写

当用户输入带有强烈自我否定的文字时，系统可将内容重写为更温和、平衡、支持性的表达。

**示例**

输入：“我今天什么都做不好。”  
输出：“你今天可能状态不太理想，但这并不代表你做得不好，你只是需要一点调整和休息。”

**价值**

相比普通建议，这种交互更有陪伴感，也更具产品特色。

---

## 8.3 微型自助干预卡片

系统根据当前情绪状态，推送一个可以在 1 分钟内完成的小行动。

**示例**

- 焦虑：做 5 次缓慢呼吸
- 生气：暂停并从 10 倒数
- 疲惫：起身活动 1 分钟并喝口水
- 难过：写下今天一件还不错的小事

**价值**

比“保持积极心态”这类泛化建议更具体、更容易执行。

---

## 8.4 情绪骤降提醒

若用户当天情绪得分明显低于近期平均值，系统会给予温和提醒，引导用户进一步记录或寻求支持。

**示例**

“你今天的状态比最近几天低一些，要不要写下原因，帮助自己理清感受？”

**价值**

在不过度医疗化的前提下，增加一定的风险感知能力。

---

## 8.5 树洞共鸣匹配

系统优先向用户展示与其当前情绪标签相似的匿名帖子。

**示例**

如果用户当前是“焦虑 + 学业压力”，则优先看到相似情绪和场景的内容。

**价值**

增强“有人和我一样”的共鸣感，而不是单纯刷社区内容。

---

## 8.6 给未来的自己写信

用户可以在某一时刻写一封只给自己的信，并设定未来日期再次打开。

**价值**

帮助用户进行情绪沉淀、自我鼓励和长期回顾。

---

## 8.7 疗愈陪伴 Agent

系统通过一个带有长期陪伴语气的 AI Agent，与用户进行多轮交流，并将最新情绪数据与上下文拼装进 Prompt，让回复更贴近用户当前状态。

**示例**

- 用户说：“我这几天一直很烦，完全静不下来。”
- Agent 结合最近 3 天“焦虑 + 学业压力 + 睡眠不足”的记录，回复更具体的共情与小行动建议

**价值**

相比单次建议卡片，Agent 让产品具备连续陪伴能力，也更容易把“记录 - 分析 - 建议”串成自然对话。

## 9. 用户流程

## 9.1 首次使用流程

1. 打开 App
2. 注册或登录
3. 查看引导页
4. 进入首页
5. 完成第一次情绪记录
6. 查看系统建议

## 9.2 日常使用流程

1. 打开 App
2. 选择当前情绪
3. 添加文字和标签
4. 保存记录
5. 查看趋势图或日历更新
6. 接收建议或自助干预卡片

## 9.3 社区使用流程

1. 打开社区页
2. 浏览匿名帖子
3. 按情绪筛选
4. 点赞或支持帖子
5. 发布自己的匿名内容

## 9.4 Agent 使用流程

1. 打开 Agent 对话页
2. 输入当前困扰、情绪或想法
3. 系统拉取用户最新情绪数据与近期趋势
4. 构建 Prompt 并调用外部中转站大模型 API
5. 返回疗愈式回复与可执行建议
6. 用户继续追问或结束对话

## 12. 技术需求

## 12.1 客户端

- Android Studio
- Java
- XML 布局
- Material Design 组件

## 12.2 本地存储

- SQLite 或 Room
- SharedPreferences 用于轻量配置存储

## 12.3 后端方案

若需要服务端支持，可选：

- Firebase Authentication
- Firebase Firestore / Realtime Database

或自建轻量 API 服务。

## 12.4 第三方库建议

- MPAndroidChart：图表展示
- RecyclerView：列表展示
- Retrofit：网络请求
- Glide / Picasso：图片加载
- Firebase SDK：身份认证与云数据支持

## 12.5 AI 实现方案

建议按以下顺序实现，降低开发风险：

1. 基于关键词词典的规则情绪分析
2. 接入情感分析 API
3. 接入 AI 生成建议与安慰重写
4. 接入心理疗愈 Agent，对接外部中转站大模型 API，并通过 Prompt 注入用户最新数据

这种分阶段实现方式，更适合作为课程项目推进。

## 13. 非功能需求

### 13.1 性能

- 页面切换响应时间应小于 2 秒
- 情绪分析结果返回时间应小于 3 秒
- 基础功能应在主流 Android 设备上流畅运行

### 13.2 易用性

- 首次使用用户应能在 1 分钟内完成第一次打卡
- 核心任务应在 3 到 4 步内完成
- 界面应清晰、平静、低认知负担

### 13.3 隐私与安全

- 情绪记录默认私密
- 匿名发帖不得暴露真实身份
- 敏感信息需安全存储与传输
- App 必须声明其不提供医学诊断
- Agent 上下文拼装时必须遵循最小必要数据原则
- 外部 API 调用前应避免传输不必要的身份标识

### 13.4 兼容性

- 支持主流 Android 版本
- 适配不同尺寸手机屏幕
- 至少支持竖屏使用

### 13.5 可靠性

- 网络失败时，核心日记功能仍可本地使用
- AI 分析失败时，系统仍应提供基础统计结果

## 15. MVP 验收标准

若满足以下条件，可认为 MVP 完成：

- 用户能够注册与登录

- 用户能够添加情绪文本与标签

- App 包含情绪日历
- 系统能够提供个性化建议
- 用户能够发起至少一轮 Agent 疗愈对话
- Agent 能基于用户最新数据生成回复，并在 API 失败时降级
- 用户能够创建和浏览匿名社区帖子
- App 可在 Android Studio 模拟器或真机上正常运行

## 16. 风险与限制

### 16.1 风险

- AI 情绪分析对复杂文本的理解可能不准确
- 若社区内容增加，审核压力会提高
- 功能扩展过多会增加开发负担
- 若表述不当，可能引发心理支持相关伦理风险

### 16.2 限制

- 开发必须控制在 Android Studio + Java 能完成的范围内
- 项目周期有限
- 第一版应优先保证稳定性与完整度，而不是功能过多

## 19. 用于展示的项目简介

MindEase 是一款面向大学生的 AI 辅助情绪日记与心理支持 App。用户可以通过它记录每日心情、分析情绪变化趋势、接收个性化建议，并在匿名树洞中表达想法。系统还将加入一个心理疗愈 Agent，在对话时结合用户最新情绪数据，通过外部大模型 API 提供更具陪伴感的回应。与普通日记 App 不同，MindEase 还加入了情绪触发因子地图、AI 安慰重写、微型自助干预卡片和树洞共鸣匹配等创新功能，帮助用户在低压力环境中提升情绪觉察与自我调节能力。

## 20. 测试模块

### 20.1 目标

- 保证 MVP 核心链路“记录 -> 分析 -> 建议”可稳定运行
- 在迭代中快速发现回归问题，降低演示风险

### 20.2 范围

- 情绪记录核心逻辑：创建、编辑、删除、最近记录查询
- 规则分析逻辑：情绪标签判定、7/30 天统计、标签频率统计
- 建议生成逻辑：基于分析结果给出可执行建议
- Agent 上下文拼装逻辑：最近记录、趋势、标签、建议的 Prompt 注入
- Agent 降级逻辑：外部 API 失败时返回本地安抚文案

### 20.3 验收标准

- 单元测试覆盖上述三类核心能力
- 单元测试覆盖 Agent Prompt 构建与降级策略
- 每次提交前可本地执行自动化测试并通过
- 关键失败场景可复现并有断言（如空数据、负向情绪聚集）
